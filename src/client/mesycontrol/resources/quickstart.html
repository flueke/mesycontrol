mesycontrol quickstart guide

There are two ways to get started: either connect to an MRC using the hardware side of the device tree or create and edit a setup using the config side.

Connecting to an MRC

Use the Add MRC connection button or the corresponding context menu entry and enter your connection details in the dialog.

Once the connection is established the busses are scanned and connected devices will appear in the hardware tree view. By double-clicking on a device or using the Open device widget button a device specific window can be opened, allowing direct editing of hardware parameters.

To create a configuration using the currently connected devices select the "Connections" node of the hardware tree and use the Copy hardware to config button. Select MRC or Device nodes to only copy the corresponding part of the tree to the configuration.

Creating a Setup

A complete configuration (possibly containing multiple MRCs) can be created and edited without requiring access to the hardware. Use the Add config button or the config-side context menu to add MRC and Device configs. Editing device parameters works the same way as for hardware devices: use the button or double-click the device to open a device specific window.

Note: If the future USB port name is not yet known you can type anything into the "USB Port" input box in the "Add MRC" dialog. The port name can later be edited by right-clicking the MRC on the config-side and selecting "Properties".

MRCs and Devices can be renamed using the context menu or pressing F2 on the selected node.

To apply configuration values to the hardware use the Apply config to hardware button. As for the hardware side subtrees can be applied by selecting the corresponding node in the config tree.

Linked mode and comparing hardware & config

Using the Toggle linked mode button linked mode can be activated. In this mode the hardware and setup trees are compared against each other, differences and conflicts are highlighted, and devices missing from either side are also shown.

In linked-mode it is possible to have changes to device parameters apply to both the hardware and the config side keeping both trees in sync. Use the Write mode and Display mode buttons in the top toolbar to control which values are displayed and where changed parameters are written to. Note: These two settings are window specific.

Use the Compare config and hardware action to re-read all hardware parameters and check the state of config vs. hardware. Required connections will be automatically established. This is a quick way of ensuring that the hardware is in the state defined by the configuration.
